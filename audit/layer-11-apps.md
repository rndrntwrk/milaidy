## Why this layer this late

- **`apps/app/src/main.tsx` is the renderer entry**, but nearly everything it imports comes from `@elizaos/app-core` (Layers 3, 7, 8). The shell is small; the work is deep in the package. So Layer 11 is **mostly a pass-through layer** by design.
- **`apps/homepage/`** has no production dependency on the running runtime — it consumes `@elizaos/app-core` only for shared CSS and a single `client-types-steward` import in `lib/cloud-api.ts`. Independent SPA.
- Catching dead test fixtures, half-wired playwright configs, and stale package.json script entries is the main yield of this layer. **None of it blocks** Phase 2/3/4.

## What to look for in this layer specifically

- **Pass-through imports vs real work.** Anything in `apps/app/src` that isn't a thin import-and-render is a layering smell (Layer 1 already flagged the inline `PhoneCompanionApp = () => null` stub on `main.tsx:21` and the 2143-LOC `vite.config.ts`).
- **Test fixtures wired but never called.** `apps/app/package.json:test:ui` references files in `test/app/` — verify the directory exists.
- **Playwright configs pointing at nonexistent test dirs.** `playwright.web-views.config.ts` says `testDir: "./test/e2e-visual"` — verify.
- **Mobile/desktop conditional sprawl.** `isMobile`, `Capacitor.getPlatform()`, `isDesktop` checks are spread across renderer; consolidation candidates surface here.
- **Marketing-site rot.** Homepage often grows release-notes, version sentinels, demo pages that ship long after the campaign ended. Audit for that.
