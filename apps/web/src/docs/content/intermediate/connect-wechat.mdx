# Connect to WeChat

WeChat is the dominant messaging and social platform in mainland China. WeChat does **not** have a public API for personal accounts, so this connector works through a third-party proxy service that emulates a WeChat client and exposes a local HTTP API. The Milady plugin then talks to that proxy.

**What you'll learn:** set up a WeChat proxy service, configure the plugin, scan the QR code.

<Callout kind="warning" title="Read this first">
This connector uses an unofficial path: a proxy service pretends to be a WeChat client on your behalf. WeChat's terms of service technically don't allow this, and WeChat does occasionally ban accounts that use it. Use a dedicated account, not your personal one, and never paste your real WeChat Pay or banking-linked account into any proxy service.
</Callout>

## What you need before you start

- **A WeChat account** you can afford to lose if it gets banned. Create a new one specifically for this.
- **A WeChat proxy service** — see the "Picking a proxy" section below.
- **A second device** (phone with WeChat installed) to scan the login QR code.
- **Milady running** with a working provider.

## Picking a proxy

You have two realistic options:

1. **Run your own proxy on a Mac or Linux server.** Open-source projects exist — search for "wechaty" or "ItChat-Puppet" on GitHub. This is the most private path but requires dev setup.
2. **Use a hosted proxy service.** Several commercial services offer this for a fee. Evaluate the provider carefully — the proxy sees every message, attachment, and contact that flows through your account.

Whichever you pick, you'll end up with:
- An **API key** the proxy service issues you
- A **proxy URL** you can reach from your Milady machine

## Step 1 — Set up the proxy service

Follow the proxy service's own documentation. At the end you should have:
- API key (paste into Milady)
- Proxy URL (e.g. `https://wechat-proxy.yourservice.com` or `http://localhost:3001` for a local one)

## Step 2 — Configure the plugin in milady.json

The WeChat connector is unusual — it's configured through `milady.json` directly, not just the plugin Configure panel, because it has structured config fields that don't fit the flat env-var UI.

Open `~/.milady/milady.json` and add a `connectors.wechat` block:

```json
{
  "connectors": {
    "wechat": {
      "apiKey": "your-proxy-api-key",
      "proxyUrl": "https://your-proxy-service/api",
      "deviceType": "ipad",
      "features": {
        "images": false,
        "groups": false
      }
    }
  }
}
```

**Field meanings:**

| Field | What it does |
| --- | --- |
| **apiKey** | Auth credential for your proxy service. Required. |
| **proxyUrl** | Base URL of your proxy service. Required. |
| **deviceType** | `ipad` (default) or `mac` — emulates what kind of WeChat client. iPad is less likely to bump you off other active sessions. |
| **features.images** | Enable image send/receive. Off by default. |
| **features.groups** | Enable group chat support. Off by default. |

<Callout kind="tip">
Keep <code>features.groups</code> off for your first run. Group support multiplies the volume of events the bot sees and makes rate limits easier to hit.
</Callout>

## Step 3 — Restart Milady and scan the QR code

<Steps>
  <li>Restart Milady (or reload the WeChat plugin from Settings → Plugins).</li>
  <li>Milady's terminal / status panel will show a QR code.</li>
  <li>Open WeChat on your phone → <strong>Me → Settings → Account &amp; Security → Manage Devices → Sign in to Web WeChat</strong>.</li>
  <li>Scan the QR code from Milady with your phone.</li>
  <li>Confirm the login on your phone.</li>
</Steps>

Milady's WeChat plugin now holds a session. The session persists — you don't need to rescan every time unless WeChat invalidates it.

## Step 4 — Test it

Send a WeChat message from another account to the account you just logged in with. Milady should see it and respond.

## Multiple accounts

If you want the agent to run multiple WeChat accounts (e.g. one for each region or purpose), use the `accounts` map:

```json
{
  "connectors": {
    "wechat": {
      "proxyUrl": "https://your-proxy/api",
      "accounts": {
        "cn-main": { "apiKey": "key1" },
        "hk-backup": { "apiKey": "key2" }
      }
    }
  }
}
```

Each account gets its own QR scan and its own session.

## Troubleshooting

**QR code scan succeeds but Milady says "not logged in."**
Usually means the proxy service didn't get a valid session back from WeChat. Check the proxy service's own logs.

**Messages arrive but agent never replies.**
Check `features.images` if the incoming message has a picture — images are off by default and the agent won't process attachments it can't decode.

**Account gets banned or locked.**
This happens. Recovery is possible through WeChat's identity verification flow on your phone, but the path out of a ban sometimes requires Chinese-language ID verification. Have a fallback account ready if this connector matters to you.
