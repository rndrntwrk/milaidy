# Connect to iMessage with BlueBubbles

Bring your Milady agent into iMessage — the blue-bubble kind — so you can text it from your iPhone, reply from your Apple Watch, or let friends chat with it from any iMessage-capable device. BlueBubbles is the bridge that makes this possible, and unlike Discord or Telegram, it does require a little bit of hardware: **a Mac you can leave on**.

**What you'll learn:** what BlueBubbles is, how to run its server on a Mac, how to point Milady at it, and what every field on the plugin settings page actually controls. Expect about 20 minutes the first time — most of it is the BlueBubbles server install, not Milady.

## What BlueBubbles is, in one paragraph

Apple doesn't give third-party apps access to iMessage. BlueBubbles gets around this by running a tiny server app on a Mac that's already signed in to iMessage — the Mac does the talking to Apple, and the BlueBubbles server exposes a local API you can hit from anywhere else. Milady talks to the BlueBubbles server, the BlueBubbles server talks to Messages.app, Messages.app talks to Apple, and the result is that your agent can send and receive real blue-bubble iMessages through your Apple ID.

## What you need before you start

- **A Mac** that is signed in to iMessage with your Apple ID, and that you can leave running and awake. A Mac mini tucked in a closet is the classic setup. Any Mac running macOS 11 (Big Sur) or newer will do.
- **Admin access on that Mac** — you'll need to grant Full Disk Access and run an installer.
- **A way to reach the Mac from Milady.** If Milady runs on the same Mac, `http://localhost:1234` is fine. If it's on a different machine on the same home network, the Mac's local IP works. If Milady runs somewhere on the wider internet, you'll need to expose the Mac via a tunnel (ngrok, Cloudflare Tunnel, Tailscale) or a port forward — covered below.
- **Milady running** with a working provider. See [Picking a provider](/docs/beginner/picking-a-provider) if you haven't done that yet.

<Callout kind="note">
iMessage will not work on a Mac that isn't signed in to iMessage. Before anything else, open Messages.app on the Mac, sign in with your Apple ID, send a test message to yourself, and confirm it shows up as a blue bubble. If it's green, iMessage isn't active and BlueBubbles has nothing to bridge.
</Callout>

## Step 1 — Install the BlueBubbles server on your Mac

<Steps>
  <li>On the Mac, open <a href="https://bluebubbles.app">bluebubbles.app</a> in a browser.</li>
  <li>Download the <strong>macOS server</strong> (not the iOS or Android app — you don't need those).</li>
  <li>Open the downloaded <code>.dmg</code> and drag <strong>BlueBubbles.app</strong> into Applications.</li>
  <li>Launch BlueBubbles from Applications. macOS may ask you to confirm because it was downloaded from the internet — click <strong>Open</strong>.</li>
  <li>On first launch, BlueBubbles walks you through a setup wizard. Follow each step as it prompts.</li>
</Steps>

During that wizard, BlueBubbles asks for a few system permissions. Grant them all — none of them are optional:

- **Full Disk Access.** iMessage stores conversations in a SQLite database under `~/Library/Messages/`. BlueBubbles reads from it. macOS blocks this by default.
- **Accessibility.** Needed so BlueBubbles can send messages through Messages.app on your behalf.
- **Notifications.** Optional but recommended — lets BlueBubbles show you when Milady sends a reply.

<Callout kind="warning" title="Full Disk Access is a big deal">
Full Disk Access lets BlueBubbles read every file the user account can see, including other chat history. Only install BlueBubbles on a Mac you own and control, and keep the BlueBubbles app updated.
</Callout>

## Step 2 — Set a server password

During the setup wizard, or from Settings inside the BlueBubbles app, you'll be asked to set a **server password**. This is the password Milady uses to authenticate — it is *not* your Apple ID password.

<Steps>
  <li>Pick a long, random password. A password manager is the right tool. Treat this like any other secret.</li>
  <li>Save it somewhere you can find it in a minute — you'll paste it into Milady next.</li>
</Steps>

<Callout kind="warning">
Anyone who knows the BlueBubbles server URL and this password can send iMessages as you. Don't reuse a password you use anywhere else, and don't expose the server to the public internet without also putting it behind a tunnel that requires its own auth (Tailscale, Cloudflare Tunnel with access policies, etc.).
</Callout>

## Step 3 — Find the server URL

By default, BlueBubbles listens on port **1234**. The URL Milady needs is:

- **Same Mac as Milady?** Use `http://localhost:1234`.
- **Different machine on the same Wi-Fi?** Use the Mac's local IP — in BlueBubbles, open Settings → Server → look for the address shown. It'll look like `http://192.168.1.42:1234`.
- **Milady runs in the cloud, or you want to reach the server from outside your network?** Use a tunnel. BlueBubbles has built-in ngrok support under Settings → Server → Proxy — enable it and BlueBubbles shows you a public `https://something.ngrok.app` URL you can copy. Alternatively, Tailscale or Cloudflare Tunnel both work fine and are more reliable for long-running setups.

<Callout kind="tip">
For most people the simplest setup is Milady and BlueBubbles on the same Mac. Everything stays on localhost, no firewall tweaking, no tunnels. You can move to a remote BlueBubbles server later without changing anything in Milady except the URL.
</Callout>

## Step 4 — Hand the credentials to Milady

<Steps>
  <li>Open Milady. Go to <strong>Settings → Plugins</strong> and find <strong>BlueBubbles</strong> in the list.</li>
  <li>Click <strong>Configure</strong>.</li>
  <li>Paste the <strong>Server URL</strong> from Step 3 into the <strong>Server URL</strong> field.</li>
  <li>Paste the <strong>Password</strong> from Step 2 into the <strong>Password</strong> field.</li>
  <li>Leave the other fields alone for your first run — defaults work.</li>
  <li>Click <strong>Save</strong>. Milady starts the BlueBubbles connection in the background.</li>
</Steps>

## Step 5 — Test it

From your iPhone (or another iMessage device on a different Apple ID), send an iMessage **to the Apple ID that's signed in on your BlueBubbles Mac**. Something simple:

> hey, are you there?

Within a second or two your Milady agent should reply in that same iMessage thread. The message should arrive as a blue bubble, on your real Apple ID, exactly like any other iMessage.

If nothing happens, walk through the troubleshooting section below.

## Every field on the BlueBubbles settings page, plain English

| Field | What it does | When to change it |
| --- | --- | --- |
| **Server URL** | Where your BlueBubbles server lives (from Step 3). | Every time the BlueBubbles server URL changes — new Mac, switched to a tunnel, moved networks. |
| **Password** | The password you set during the BlueBubbles wizard. | If you rotate it in BlueBubbles, update here. |
| **Webhook path** | The URL path Milady listens on for incoming messages from the BlueBubbles server. | Leave default. Only change if you're running multiple Milady instances behind a shared reverse proxy. |
| **DM policy** | Who's allowed to DM your agent. `allow-all` lets anyone who texts your Apple ID get a reply. `allow-from` restricts replies to the list below. | Turn on `allow-from` the moment your number gets shared more widely. |
| **Allow from** | The list of phone numbers or email addresses that are allowed to DM when `DM policy` is `allow-from`. Comma-separated. | Use the same format that shows up in Messages.app — `+15551234567` or `friend@example.com`. |
| **Group policy** | Same as DM policy but for group chats. | Groups are noisy — consider `allow-from` even if your DM policy is `allow-all`. |
| **Group allow from** | The group IDs the agent is allowed to participate in. See "Getting group IDs" below. | Whenever you add a new group. |
| **Send read receipts** | If on, Milady marks incoming messages as read in Messages.app so the sender sees the "read" indicator. | Off if you want your agent to be stealthier. On if you want it to feel like a real conversation partner. |

## Getting group IDs

If you use `allow-from` for groups, you need the ID of each group you want your agent in. The easiest way:

<Steps>
  <li>Open the BlueBubbles Mac app and go to <strong>Devices → Chats</strong>.</li>
  <li>Find the group in the list. Click it.</li>
  <li>Copy the <strong>GUID</strong> shown in the details pane. It's a long string starting with <code>iMessage;+;</code>.</li>
  <li>Paste it into Milady's <strong>Group allow from</strong> field, comma-separated if you have more than one.</li>
</Steps>

## Troubleshooting

**Milady says "BlueBubbles not reachable" or the status is red.**
The server URL is wrong, the BlueBubbles server isn't running, or something (firewall, VPN) is blocking the connection between Milady and the Mac. Open a browser on the Milady machine and visit the server URL — you should see the BlueBubbles server's health endpoint respond. If that fails, Milady will fail too.

**Server is reachable but no messages come through.**
The webhook isn't firing. Open BlueBubbles on the Mac → Settings → Server → Webhooks and confirm Milady's webhook URL is listed. If it isn't, Milady couldn't register it — most commonly because the BlueBubbles server can't reach Milady back. This is the same problem in reverse: BlueBubbles and Milady need to be able to reach each other, not just Milady-to-BlueBubbles.

**Messages arrive but replies go out as green bubbles.**
This is iMessage telling you it couldn't send through Apple's network and fell back to SMS. Check Messages.app on the Mac — if it's showing a "Sign in to iMessage" banner, that's your culprit. Sign back in, send one manual message from Messages.app to confirm it's blue again, then retry.

**Everything works but group messages get a green "delivered" and then silence.**
Group policy is probably set to `disabled` or the group isn't in the allow list. Check Step 4's table and update.

**Replies show as "from Milady" instead of from me.**
They shouldn't — BlueBubbles sends as the Apple ID signed in on the Mac, which is you. If the sender name looks wrong, check the display name on that Apple ID in System Settings → Apple ID → Name, Phone, Email.

## Privacy and safety

- **Your iMessages stay on your Mac and your Milady machine.** Milady never uploads your message history anywhere on its own. Whatever cloud model provider you picked in [Picking a provider](/docs/beginner/picking-a-provider) is the one exception — any message your agent answers is sent to that provider as part of the prompt.
- **BlueBubbles is an independent project**, not made by Milady and not made by Apple. Its source is on [GitHub](https://github.com/BlueBubblesApp/bluebubbles-server). Keep it updated.
- **Full Disk Access is powerful.** On the same Mac, any app with it can read any file you can read. BlueBubbles is the only app that should have it for this use case; don't pile on.

## You're done when…

- You can text the Mac's Apple ID from your iPhone and get a blue-bubble reply from your Milady agent within a couple of seconds.
- The BlueBubbles plugin status indicator in Milady shows green.
- Group chats you've allow-listed also get replies; group chats you haven't don't.
