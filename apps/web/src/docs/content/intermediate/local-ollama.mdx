# Run Milady on a local model with Ollama

Ollama is the easiest way to run a language model entirely on your own machine. No API keys, no data leaving your computer, no usage bills. The tradeoff is that it needs a reasonably capable Mac, PC, or Linux box — and the models are smaller than what the big cloud providers can give you.

**What you'll learn:** install Ollama, download a model, point Milady at it. About ten minutes if your internet is fast.

## What you need before you start

- **A computer that can handle it.** Rough rule of thumb: 16 GB RAM minimum to run a small model comfortably, 32+ GB if you want a mid-sized model. A GPU helps a lot on Linux and Windows; Apple Silicon Macs run it well without one.
- **About 5 GB of free disk space** for a small model. Larger models want 20–40 GB each.
- **Milady installed and running.**

<Callout kind="note">
Ollama runs as a local HTTP server on your machine. Nothing is sent to the internet once a model is downloaded. This is the most private way to run Milady.
</Callout>

## Step 1 — Install Ollama

<Steps>
  <li>Open <a href="https://ollama.com/download">ollama.com/download</a>.</li>
  <li>Download the installer for your OS — macOS, Windows, or Linux.</li>
  <li>Run the installer. On macOS it lives in Applications; on Windows it runs as a background service; on Linux it installs as a systemd service.</li>
  <li>Open a terminal and run <code>ollama --version</code> to confirm it's installed. You should see a version number.</li>
</Steps>

## Step 2 — Pull a model

Models don't come bundled — you download the one you want.

<Steps>
  <li>In a terminal, run <code>ollama pull llama3.2</code>. This downloads Meta's Llama 3.2 model, a good middle-of-the-road starting point. Expect a few gigabytes.</li>
  <li>When it's done, run <code>ollama run llama3.2</code> and ask it something to confirm it works. Type <code>/bye</code> to exit.</li>
</Steps>

**Other good starter models:**

| Model | Pull command | Size | Notes |
| --- | --- | --- | --- |
| Small, fast | `ollama pull llama3.2:3b` | ~2 GB | Runs on almost anything. Responses are short but quick. |
| Balanced | `ollama pull llama3.2` | ~4.7 GB | Default. Good starting point. |
| Larger, smarter | `ollama pull llama3.3:70b` | ~40 GB | Needs a beefy machine. Much better quality. |
| Embeddings | `ollama pull nomic-embed-text` | ~270 MB | For Milady's knowledge features. Pull this alongside a chat model. |

## Step 3 — Point Milady at Ollama

<Steps>
  <li>Open Milady. Go to <strong>Settings → Plugins → Ollama → Configure</strong>.</li>
  <li>Leave <strong>Base URL</strong> as <code>http://localhost:11434</code> — that's the default port Ollama listens on.</li>
  <li>Set <strong>Small model</strong>, <strong>Medium model</strong>, and <strong>Large model</strong> to the model tag you pulled (e.g. <code>llama3.2:3b</code>, <code>llama3.2</code>, <code>llama3.3:70b</code>). If you only pulled one, use it for all three.</li>
  <li>Optionally set <strong>Embedding model</strong> to <code>nomic-embed-text</code> if you pulled it.</li>
  <li>Click <strong>Save</strong>.</li>
</Steps>

## Step 4 — Test it

Start a new chat in Milady. Send a message. Your reply is now coming from a model running on your own machine.

If it feels slow: you're running a model too big for your hardware. Swap to a smaller tag (`llama3.2:3b` or similar) and it'll feel much snappier.

## Troubleshooting

**"Connection refused" or Milady can't reach Ollama.**
Ollama isn't running. On macOS, check the menu bar for the Ollama icon. On Linux, run `systemctl status ollama`. On Windows, check Task Manager for `ollama.exe`.

**Replies are very slow or your machine fans spin up hard.**
The model is too large for your hardware. Switch to a smaller tag — `llama3.2:3b` runs on almost anything.

**Replies are gibberish or wildly off-topic.**
The model you picked is too small for the task. Try a larger tag if your hardware can handle it.
