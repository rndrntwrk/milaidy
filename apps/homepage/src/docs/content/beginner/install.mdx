# Install Milady

Milady ships as a desktop app for macOS, Windows, and Linux. There's also a CLI install for people who want to run it from a terminal — both options use the same underlying runtime, so pick whichever fits how you work.

**Time needed:** about five minutes on a fresh machine, plus a few minutes for the first-run setup.

## Which version should I download?

Milady is published in two release channels, plus the source code on GitHub. Pick the one that matches how much risk you're comfortable with — you can switch later.

| Channel | Who it's for | Link |
| --- | --- | --- |
| **Stable** | Everyone, by default. Tested, notarized, and safe to leave running day to day. | [Latest stable release](https://github.com/milady-ai/milady/releases/latest) |
| **Canary** | Early adopters who want new features the day they land and don't mind the occasional rough edge. | [All releases (look for the *Pre-release* badge)](https://github.com/milady-ai/milady/releases) |
| **Source (GitHub)** | Developers who want to read the code, file issues, or build from `main` themselves. | [github.com/milady-ai/milady](https://github.com/milady-ai/milady) |

**Stable** is what the big "Download" buttons on [milady.ai](https://milady.ai) point to. Release tags look like `v1.4.0` — no suffix. Bug fixes and security patches land here first.

**Canary** is tagged with `alpha`, `beta`, `rc`, or `nightly` (e.g. `v1.5.0-beta.2`). New connectors, UI experiments, and plugin system changes show up here days or weeks before they promote to stable. If you hit a bug on canary, the fastest way to get it fixed is to [file an issue](https://github.com/milady-ai/milady/issues) with the exact tag — canary users are how we catch regressions before stable ships. Canary installs live alongside stable on macOS as `Milady-canary.app`, so you can run both side-by-side without blowing away your stable config.

**Source (GitHub)** isn't a build — it's the full codebase. Use it if you want to read how something works, contribute a fix, or build Milady yourself from a branch. The README has build instructions. Everything else in these docs applies whether you installed from stable, canary, or a source build.

<Callout kind="tip">
Not sure? Pick **stable**. You can always switch to canary later by downloading it from the releases page; the two channels share the same `~/.milady/` config directory, so your characters, keys, and conversations come with you.
</Callout>

## Desktop app (recommended)

The desktop app is the friendliest way to get started. It bundles the runtime, the dashboard, and the companion UI into one installable package with auto-updates.

Grab the right file from the [latest GitHub release](https://github.com/milady-ai/milady/releases/latest):

| Platform | File |
| --- | --- |
| macOS (Apple Silicon — M1/M2/M3/M4) | `Milady-arm64.dmg` |
| macOS (Intel — pre-2020 Macs) | `Milady-x64.dmg` |
| Windows 10 and 11 | `Milady-Setup.exe` |
| Linux (AppImage, any distro) | `Milady.AppImage` |
| Linux (Debian / Ubuntu) | `milady_*.deb` |
| iOS | [App Store](https://apps.apple.com/app/milady-private-ai-assistant/id0000000000) |
| Android | [Google Play](https://play.google.com/store/apps/details?id=ai.milady.app) or APK on GitHub |

<Callout kind="note">
Every desktop build is code-signed and (on macOS) notarized. You shouldn't see a Gatekeeper "unidentified developer" warning. If you do, you downloaded a build from somewhere that isn't `github.com/milady-ai/milady` — delete it and grab the official one.
</Callout>

### macOS

<Steps>
  <li>Download the `.dmg` that matches your Mac's chip (Apple Silicon or Intel).</li>
  <li>Double-click to mount. Drag `Milady.app` into your `Applications` folder.</li>
  <li>Open it from Launchpad or Spotlight. The first launch kicks off onboarding — see <a href="/docs/beginner/first-run">Your first launch</a>.</li>
</Steps>

### Windows

<Steps>
  <li>Download `Milady-Setup.exe`.</li>
  <li>Run the installer. Accept the SmartScreen prompt if it appears — the installer is signed but SmartScreen can be cautious with new publishers.</li>
  <li>Milady lands in your Start menu. Open it and follow the first-launch flow.</li>
</Steps>

### Linux

Pick AppImage (works everywhere) or the `.deb` (for Debian, Ubuntu, Pop!_OS, and the like).

<Steps>
  <li>Download the `.AppImage` or `.deb`.</li>
  <li>**AppImage:** `chmod +x Milady.AppImage` and double-click (or run from a terminal). **.deb:** `sudo dpkg -i milady_*.deb`.</li>
  <li>Launch from your application menu or by running `milady-desktop` in a terminal.</li>
</Steps>

## CLI install (power users)

If you'd rather run Milady from a terminal — on a headless Linux box, a remote server, or just because you prefer the CLI — there's a one-line installer.

**macOS / Linux / WSL / Git Bash:**

```bash
curl -fsSL https://get.milady.ai | bash
```

**Windows PowerShell:**

```powershell
irm https://get.milady.ai/install.ps1 | iex
```

The installer checks for Node.js 22 or newer, offers to install it if missing, then installs Milady globally via npm. When it finishes, run:

```bash
milady start
```

…and the runtime comes up. Your desktop browser opens the dashboard at `http://localhost:31337`.

<Callout kind="tip">
If you already have a desktop app installed and you're also using the CLI, that's fine — they share the same config directory at `~/.milady/`. Whichever you launch last wins the shared state, so don't run both at the same time.
</Callout>

## Verify it worked

After install, Milady creates a folder at `~/.milady/` (on Windows: `%USERPROFILE%\.milady\`). That's where your settings, conversations, uploaded documents, and character data live. If that folder doesn't exist after a successful install, something went wrong — see [Troubleshooting](/docs/beginner/troubleshooting).
