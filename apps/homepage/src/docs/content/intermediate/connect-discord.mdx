# Connect to Discord

Bring your Milady agent into a Discord server so it can respond to messages and mentions the same way you chat with it on your desktop. This entire walkthrough happens inside Settings — no code, no terminal.

**What you'll learn:** how to create a Discord bot, invite it to a server, hand the token to Milady, and verify it's working. Takes about ten minutes the first time.

## What you need before you start

- **A Discord account** with permission to add bots to at least one server. If it's your own server, you're set. If it's someone else's, you'll need their permission and the "Manage Server" role on that server.
- **Milady running on your machine** with a working provider (see [Picking a provider](/docs/beginner/picking-a-provider)).

## Step 1 — Create a Discord application

Discord calls your bot an "application" in their developer portal. You'll create one, add a bot user to it, and grab two values: a **bot token** and an **application ID**.

<Steps>
  <li>Open <a href="https://discord.com/developers/applications">discord.com/developers/applications</a> and sign in.</li>
  <li>Click <strong>New Application</strong>. Give it a name — this is what people will see as the bot's display name. You can change it later.</li>
  <li>Once the app is created, copy the <strong>Application ID</strong> from the General Information page. You'll need it in a minute.</li>
  <li>Click the <strong>Bot</strong> tab in the left sidebar.</li>
  <li>Click <strong>Reset Token</strong> and copy the token that appears. <strong>You'll only see this once</strong> — if you navigate away, you have to reset it again.</li>
</Steps>

<Callout kind="warning">
The bot token is a secret. Treat it like a password. Never paste it in a screenshot, never check it into git, never share it in a Discord message "to help someone troubleshoot." Anyone with the token can speak as your bot.
</Callout>

## Step 2 — Enable Message Content Intent

Discord requires bots to explicitly opt in to reading message content. Without this, your Milady agent can't see what people are saying.

<Steps>
  <li>Still on the Bot tab, scroll down to <strong>Privileged Gateway Intents</strong>.</li>
  <li>Toggle <strong>Message Content Intent</strong> on.</li>
  <li>Save changes.</li>
</Steps>

## Step 3 — Invite the bot to your server

<Steps>
  <li>Click the <strong>OAuth2</strong> tab → <strong>URL Generator</strong>.</li>
  <li>Under <strong>Scopes</strong>, check <strong>bot</strong> and <strong>applications.commands</strong>.</li>
  <li>Under <strong>Bot Permissions</strong>, check at minimum: <strong>Send Messages</strong>, <strong>Read Messages/View Channels</strong>, <strong>Use Slash Commands</strong>. Add anything else you want the bot to do (Embed Links, Attach Files, Add Reactions are all useful).</li>
  <li>Copy the <strong>Generated URL</strong> at the bottom and open it in your browser.</li>
  <li>Discord asks which server to add the bot to. Pick one and confirm.</li>
</Steps>

Your bot now appears in the server's member list with a "Bot" badge. It'll show as offline until Milady actually connects to it.

## Step 4 — Hand the credentials to Milady

<Steps>
  <li>Open Milady. Go to <strong>Settings → Connectors → Discord</strong>.</li>
  <li>Paste the <strong>Bot Token</strong> into the token field.</li>
  <li>Paste the <strong>Application ID</strong> into the application ID field.</li>
  <li>(Optional but recommended) Under <strong>Channel IDs</strong>, list the specific channels you want the bot to read and respond in. If you leave it blank, the bot listens in every channel it has access to — which can get noisy fast.</li>
  <li>Click <strong>Save</strong> or <strong>Connect</strong>. Milady restarts the Discord connection in the background.</li>
</Steps>

Within a few seconds, the bot in your server flips from offline to online.

## Getting channel IDs

Discord hides channel IDs by default. To grab one:

<Steps>
  <li>In Discord, open <strong>User Settings → Advanced</strong> and enable <strong>Developer Mode</strong>.</li>
  <li>Right-click any channel name. You'll see a new <strong>Copy Channel ID</strong> option at the bottom of the menu.</li>
  <li>Paste that ID into Milady's channel list (comma-separated if you want multiple).</li>
</Steps>

## Step 5 — Test it

Go to one of the channels your bot is listening in and send a message that mentions the bot directly:

> @YourBotName hey, are you there?

You should see the bot start typing within a second or two, then reply with whatever your character's voice sounds like. If you don't get a response:

- **Bot shows offline?** The token is wrong. Double-check you copied it completely — no leading/trailing whitespace, no quotes.
- **Bot is online but doesn't respond?** Message Content Intent is probably off, or the bot doesn't have Read Messages permission in that specific channel. Check channel-level permissions in Discord.
- **Bot responds but says something weird like "I can't see your message"?** Same as above — usually Message Content Intent.

## Useful options

In Settings → Connectors → Discord, you'll see a few toggles worth knowing about:

| Option | What it does |
| --- | --- |
| **Only respond to mentions** | Bot only replies when someone `@` mentions it. Off by default — the bot will respond to any message in its listening channels. Turn on in busy servers. |
| **Ignore direct messages** | Off by default. Turn on if you don't want the bot to reply to DMs to itself. |
| **Ignore bot messages** | On by default. Prevents bot-to-bot loops when another bot is in the same channel. |
| **Listen-only channels** | Channels the bot reads but doesn't send unsolicited messages in. Useful for letting your agent build context from a channel without being chatty in it. |

## What actually happens when a message comes in

For the curious: when someone sends a message in a channel your bot is listening to, Discord pushes it over a WebSocket to Milady. Milady checks that the bot should respond (based on the filters above), builds a prompt that includes the incoming message and recent conversation history, sends it to your language model provider, streams the response back, and posts it to the channel. The whole round trip usually takes 1–3 seconds with a fast provider.

<Callout kind="tip">
Your agent has a different memory per channel by default. Conversations in #general don't leak into conversations in #dev-chat. If you want shared context across channels, that's configurable in Settings → Memory.
</Callout>
