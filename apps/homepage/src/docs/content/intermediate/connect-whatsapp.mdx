# Connect to WhatsApp

WhatsApp has **two very different modes** and picking the right one matters. Read this page carefully before starting — they have different setup effort, different reliability, and different risk profiles.

## Pick your mode

**Cloud API (recommended for anything serious):** use Meta's official WhatsApp Business Platform. Requires a business, a verified phone number, and app approval from Meta. More work up front, works reliably forever.

**Baileys (for tinkerers):** uses an unofficial API that scans a QR code to log in as your personal WhatsApp account. Works in five minutes, but **violates WhatsApp's terms of service** and can get your personal account banned — sometimes immediately, sometimes after weeks.

The rest of this page has two halves — scroll to the mode you picked.

---

## Mode 1 — Cloud API (official)

### What you need

- **A Meta Business account** and a real business entity (even a sole proprietorship works).
- **A phone number** that's not currently registered on WhatsApp. You'll register it through Meta.
- **Facebook Developer account** at <a href="https://developers.facebook.com">developers.facebook.com</a>.
- **Milady on a public URL** (Cloud API pushes messages via webhook — see tunneling options below).

### Step 1 — Create the Meta app

<Steps>
  <li>Open <a href="https://developers.facebook.com/apps">developers.facebook.com/apps</a> and click <strong>Create App</strong>.</li>
  <li>Pick <strong>Business</strong> as the use case.</li>
  <li>Give it a name and link it to your Meta Business account.</li>
</Steps>

### Step 2 — Add WhatsApp to the app

<Steps>
  <li>In the new app's dashboard, click <strong>Add Product</strong> → find <strong>WhatsApp</strong> → <strong>Set up</strong>.</li>
  <li>Follow Meta's guided flow to associate a phone number. You can use the test number Meta provides for free during development, then add your real number later.</li>
  <li>Copy the <strong>Phone Number ID</strong> and <strong>WhatsApp Business Account ID</strong> shown on the API Setup page.</li>
</Steps>

### Step 3 — Generate a permanent access token

<Steps>
  <li>In Meta Business Settings → Users → System Users → Add.</li>
  <li>Create a system user, assign it to the WhatsApp Business Account with full permissions.</li>
  <li>Generate a token for it with <code>whatsapp_business_messaging</code> and <code>whatsapp_business_management</code> scopes. Never-expiring is the right option for production use.</li>
  <li>Copy the token.</li>
</Steps>

### Step 4 — Set up the webhook

<Steps>
  <li>Back in the app dashboard → WhatsApp → Configuration.</li>
  <li>Callback URL: <code>https://&lt;your-public-milady-url&gt;/webhooks/whatsapp</code></li>
  <li>Verify token: pick any string — this is just used to verify the handshake. Save it, you'll paste it into Milady too.</li>
  <li>Meta pings the URL to verify. Milady must be running and reachable for this to succeed.</li>
  <li>Subscribe to <strong>messages</strong> and <strong>message_status</strong> events.</li>
</Steps>

### Step 5 — Hand credentials to Milady

<Steps>
  <li>Settings → Plugins → WhatsApp → Configure.</li>
  <li>Set <strong>Mode</strong> to <code>cloud</code>.</li>
  <li>Paste the <strong>Access token</strong> from Step 3.</li>
  <li>Paste the <strong>Phone Number ID</strong> and <strong>Business Account ID</strong> from Step 2.</li>
  <li>Paste the <strong>Webhook verify token</strong> you chose in Step 4.</li>
  <li>Save.</li>
</Steps>

### Step 6 — Test

Send a WhatsApp message to the registered business number from a personal phone. Milady should reply within a couple of seconds.

## Mode 2 — Baileys (personal, QR code, ToS-violating)

### What you need

- **A personal WhatsApp account** you can afford to lose. Never use your main number.
- **A persistent directory** for session files so you don't have to re-scan the QR code after every restart.
- **Milady running** with a working provider.

### Step 1 — Pick a session directory

Pick an absolute path on your machine, e.g. `~/.milady/whatsapp-auth/`. Milady will store the encrypted session here.

### Step 2 — Configure the plugin

<Steps>
  <li>Settings → Plugins → WhatsApp → Configure.</li>
  <li>Set <strong>Mode</strong> to <code>baileys</code>.</li>
  <li>Set <strong>Auth dir</strong> to the absolute path from Step 1.</li>
  <li>Save.</li>
</Steps>

### Step 3 — Scan the QR code

<Steps>
  <li>Milady shows a QR code in its console or status panel.</li>
  <li>On your phone, open WhatsApp → Settings → Linked Devices → Link a Device → scan the code.</li>
  <li>Milady is now connected. The session file is saved to your auth dir for reuse.</li>
</Steps>

### Step 4 — Test

Send yourself a message from another WhatsApp account. Milady should see it and reply.

<Callout kind="danger" title="Baileys bans">
Meta actively detects and bans unofficial WhatsApp clients. Timelines vary — some users run Baileys for years, some get banned in hours. There is no appeal. Your phone number is tied to your WhatsApp account; if it's banned, that number cannot register again for weeks or months.
</Callout>

## Troubleshooting

**Cloud API: webhook verification fails during setup.**
Milady must be running and publicly reachable at the URL you entered. The verify token must match on both sides exactly.

**Cloud API: messages arrive but replies go nowhere.**
Check the access token hasn't expired (for system user tokens this shouldn't happen, but double-check the scopes include `whatsapp_business_messaging`).

**Baileys: QR code doesn't scan.**
The code expires after 20 seconds. Restart the plugin — it'll generate a new one.

**Baileys: "Connection closed" errors after being stable.**
Usually means the session was invalidated from the phone side (another device linked, or WhatsApp detected the login). Delete the auth dir and rescan.
