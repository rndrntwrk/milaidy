# Connect to Zalo

Zalo is Vietnam's dominant messaging and social platform. This connector uses the **official Zalo Official Account API** — the one Zalo blesses for businesses and app developers. For a personal-account path with different tradeoffs, see [Connect to Zalo (personal)](/docs/intermediate/connect-zalouser).

**What you'll learn:** create a Zalo Official Account, register an app in the Zalo developer portal, generate OAuth tokens, and point Milady at them.

## What you need before you start

- **A Zalo account** (personal) — you'll use it to sign in to the developer portal.
- **A Zalo Official Account (OA)** — you can create one at <a href="https://oa.zalo.me">oa.zalo.me</a>. For serious use it helps to have a Vietnamese business entity, but individual OAs are also possible.
- **A publicly reachable HTTPS URL** for Milady (Zalo pushes messages via webhook).
- **Milady running** with a working provider.

## Step 1 — Create a Zalo Official Account

<Steps>
  <li>Open <a href="https://oa.zalo.me">oa.zalo.me</a> and sign in with your personal Zalo account.</li>
  <li>Follow the OA creation flow. Pick a category appropriate for what the bot will do.</li>
  <li>Note the OA's ID.</li>
</Steps>

## Step 2 — Register an app in the Zalo developer portal

<Steps>
  <li>Open <a href="https://developers.zalo.me">developers.zalo.me</a> and sign in.</li>
  <li>Create a new app. Link it to the OA you created in Step 1.</li>
  <li>In the app settings, copy the <strong>App ID</strong> and <strong>Secret Key</strong>.</li>
</Steps>

## Step 3 — Get OAuth access + refresh tokens

Zalo's Official Account API uses OAuth. You authorize your OA once, and the developer portal gives you an access token (short-lived) and a refresh token (long-lived).

<Steps>
  <li>In the developer portal, go to your app → <strong>API Explorer</strong> or the OAuth flow page.</li>
  <li>Authorize your OA. The portal returns an access token and a refresh token.</li>
  <li>Copy both. Milady will use the refresh token to get new access tokens automatically once the current one expires (every few hours).</li>
</Steps>

## Step 4 — Set up the webhook

<Steps>
  <li>In the Zalo developer portal, set the app's <strong>Webhook URL</strong> to <code>https://&lt;your-public-milady-url&gt;/webhooks/zalo</code>.</li>
  <li>Zalo verifies the URL — Milady must be running and reachable.</li>
  <li>Subscribe to the events you need: incoming messages, user follow/unfollow, etc.</li>
</Steps>

## Step 5 — Hand credentials to Milady

<Steps>
  <li>Open Milady. Go to <strong>Settings → Plugins → Zalo → Configure</strong>.</li>
  <li>Paste the <strong>App ID</strong> and <strong>Secret Key</strong> from Step 2.</li>
  <li>Paste the <strong>Access token</strong> and <strong>Refresh token</strong> from Step 3.</li>
  <li>Click <strong>Save</strong>.</li>
</Steps>

## Step 6 — Test

From another Zalo account, message your OA. Within a few seconds Milady should reply.

## Troubleshooting

**"Invalid access token" after a few hours.**
Access tokens expire and need refreshing. Milady does this automatically if the refresh token is correct. If it's not, you'll need to re-run the OAuth flow in Step 3.

**Webhook verification fails.**
Milady's public URL isn't reachable, or the tunnel is down.

**Messages arrive but replies never send.**
OA might not have the right scopes. Check the Zalo developer portal → app permissions and make sure Messaging is approved.
