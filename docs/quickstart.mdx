On first launch, Milady asks for:

1. where the runtime should live
2. which character to use
3. which provider should answer messages

## 1. Pick the runtime location

| Option | Use it when |
|--------|-------------|
| `Local` | Milady should run on this machine |
| `LAN` | the runtime already exists on another machine in your network |
| `Remote` | the runtime already exists on a VPS, homelab box, or private URL |
| `Eliza Cloud` | you want a managed backend |

If you are not sure, pick `Local`.

## 2. Pick the character

Choose a name and a starting character. This sets the initial personality, voice, and avatar.

You can change all of it later in [Change Your Character](/user/change-character).

## 3. Connect a provider

Choose one of these:

- `Cloud API`
- `Ollama`
- `Eliza Cloud`

Milady tests the connection before continuing.

## 4. Send a test message

After setup, send a short message and confirm:

- the agent responds
- the speed is acceptable
- the name and tone are right
- voice works if you want speech enabled
