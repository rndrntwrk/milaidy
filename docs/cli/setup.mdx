# `milady setup`

Walks through initial provider keys and core options when you are onboarding a new machine or state directory. Bootstraps the agent workspace and runs a health check at the end.

## When to use

- Fresh install before first `milady start` or dashboard use.
- You need to add API keys without editing JSON by hand.

## Flags

| Flag | Description |
|------|-------------|
| `--workspace <dir>` | Set the agent workspace directory. |
| `--provider <name>` | Model provider name for non-interactive setup (e.g. `anthropic`, `openai`, `ollama`). |
| `--key <value>` | API key or URL to store for the selected provider. Prefer `--key-stdin` to avoid shell history exposure. |
| `--key-stdin` | Read the API key or URL from stdin instead of passing it as an argument. |
| `--no-wizard` | Skip the interactive model provider wizard. |

## Examples

```bash
# Interactive setup
milady setup

# Non-interactive: set Anthropic key via stdin
echo "sk-ant-..." | milady setup --provider anthropic --key-stdin

# Non-interactive: set Ollama URL
milady setup --provider ollama --key http://localhost:11434 --no-wizard
```
