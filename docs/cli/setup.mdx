# `milady setup`

Walks through initial provider keys, character selection, and core options when onboarding a new machine or state directory. This is the recommended first command after installing Milady.

## When to use

- Fresh install before first `milady start`
- Setting up a new state directory
- Adding API keys without editing JSON by hand

## What it configures

1. **Runtime location** — Local, LAN, Remote, or Eliza Cloud
2. **Character** — Name and starting personality
3. **Provider** — Cloud API key, Ollama, or Eliza Cloud connection
4. **Connection test** — Verifies the provider responds before finishing

## Examples

```bash
# Interactive setup (recommended)
milady setup

# Show available flags
milady setup --help
```

## After setup

```bash
# Start the runtime
milady start

# Open the web dashboard
milady dashboard
```

Your configuration is saved to `~/.milady/milady.json`. Edit it directly for advanced options, or run [`milady configure`](/cli/configure) to review common settings.
