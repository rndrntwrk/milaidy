# `milady configure`

Displays a quick reference of common configuration options, environment variables, and active provider status. This is a read-only guide, not a setup wizard — use [`milady setup`](/cli/setup) for first-time onboarding.

## What it shows

- Which model providers are detected and their status
- Common environment variables and where to set them
- Current config file path and state directory
- Suggested next steps based on your configuration

This command does not accept flags. For reading and writing config values, use [`milady config`](/cli/config). For interactive key setup, use [`milady setup`](/cli/setup).

## When to use

- You need a quick reminder of which env var activates a provider
- You want to confirm which providers are currently detected
- You are not sure where your config file lives

For interactive key entry and first-time setup, use [`milady setup`](/cli/setup) instead.
