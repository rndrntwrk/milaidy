# `milady doctor`

Runs health checks grouped by category: **System** (runtime version, node_modules, build artifacts, local upstreams), **Configuration** (config file, model API key, host binding), **Storage** (state directory, database, disk space), and **Network** (port availability).

## What it checks

## Flags

| Flag | Description |
|------|-------------|
| `--no-ports` | Skip port availability checks. |
| `--fix` | Automatically fix issues where possible (runs auto-fixable `milady` sub-commands). |
| `--json` | Output results as JSON (CI-friendly). Exits with code 1 if any check fails. |

## Examples

```bash
milady doctor
milady doctor --fix
milady doctor --json
milady doctor --no-ports
```

From a source checkout:

```bash
bun run milady:doctor
```

## When to use it

- After installing or updating Milady
- After changing ports or state directory paths
- After moving `~/.milady/` to a new location
- When the agent fails to start or connectors do not load
- When filing a bug report — include the doctor output
