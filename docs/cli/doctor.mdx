# `milady doctor`

Runs a full environment health check, flagging missing dependencies, port conflicts, and configuration issues.

## What it checks

- **Node.js / Bun version** — verifies Node.js 22+ and Bun are available
- **Config file** — checks that `~/.milady/milady.json` exists and is valid JSON
- **State directory** — verifies `~/.milady/` is accessible and writable
- **Port availability** — checks whether default ports (2138, 18789, 31337) are free or already in use
- **Model providers** — reports which API keys are configured and reachable
- **Database** — checks that the SQLite/PGlite database is accessible

For maintainers working from a source checkout, `doctor` also checks the repo-local vendored workspace under `./eliza` and reports whether `@elizaos/core` resolves to vendored source or a published install.

## Usage

```bash
milady doctor
milady doctor --help
```

From a source checkout:

```bash
bun run milady:doctor
```

## When to use it

- After installing or updating Milady
- After changing ports or state directory paths
- After moving `~/.milady/` to a new location
- When the agent fails to start or connectors do not load
- When filing a bug report — include the doctor output
