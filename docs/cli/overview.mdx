# CLI overview

The `milady` binary (also published as `milady-ai` / `miladyai`) is the primary way to start the API server, run setup wizards, and manage local configuration from a terminal.

## Requirements

- **Node.js 22+** or **Bun** as documented in [Installation](/installation).
- Config and state live under the Milady state directory (by default `~/.milady/` when `ELIZA_NAMESPACE=milady`).

## Command map

| Command | Purpose |
|---------|---------|
| [`milady start`](/cli/start) | Start the elizaOS agent runtime (`milady run` is an alias) |
| [`milady setup`](/cli/setup) | First-time wizard — provider keys, workspace bootstrap |
| [`milady configure`](/cli/configure) | Configuration guidance |
| [`milady config`](/cli/config) | Config helpers (`get`, `path`, `show`) |
| [`milady dashboard`](/cli/dashboard) | Open the Control UI in your browser |
| [`milady models`](/cli/models) | Show configured model providers |
| [`milady plugins`](/cli/plugins) | Browse, install, and manage plugins |
| [`milady update`](/cli/update) | Check for and install updates |
| [`milady doctor`](/cli/doctor) | Health and environment checks |
| [`milady db`](/cli/db) | Database management |

## Interactive TUI vs server-only mode

Running `milady` with no subcommand launches the interactive terminal UI with a live activity log, token/cost counters, and hotkeys:

| Key | Action |
|-----|--------|
| `q` | Quit gracefully |
| `r` | Restart gateway |
| `d` | Open dashboard in browser |
| `l` | Toggle log view |
| `c` | Compact/clear activity |
| `?` | Show help |
| `↑/↓` | Scroll activity |

Running `milady start` runs in server-only mode — logs go to stdout/stderr with no interactive UI. Use this for daemonized deployments or when running under a process manager.

## Global flags

| Flag | Purpose |
|------|---------|
| `--verbose` | Enable informational runtime logs |
| `--debug` | Enable debug-level runtime logs |
| `--version` | Print version and exit |
| `--help` | Print help |

## Help

Run `milady --help` or `milady <command> --help` for the exact flags in your build. Implementation lives under `eliza/packages/app-core/src/cli/` in this repository.
