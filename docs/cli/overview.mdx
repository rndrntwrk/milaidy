# CLI overview

The `milady` binary (also published as `milady-ai` / `miladyai`) is the primary way to start the API server, run setup wizards, and manage local configuration from a terminal.

## Requirements

- **Node.js 22+** or **Bun** as documented in [Installation](/installation).
- Config and state live under the Milady state directory (by default `~/.milady/` when `ELIZA_NAMESPACE=milady`).

## Global Flags

| Flag | Description |
|------|-------------|
| `--verbose` | Enable informational runtime logs |
| `--debug` | Enable debug-level runtime logs |
| `--version` | Print version and exit |
| `--help` | Show help for any command |

## Command map

| Command | Purpose |
|---------|---------|
| [`milady start`](/cli/start) | Run the API server (embedded runtime). `milady run` is an alias. |
| [`milady setup`](/cli/setup) | First-time wizard (keys, workspace) |
| [`milady configure`](/cli/configure) | Configuration guidance (prints env-var help) |
| [`milady config`](/cli/config) | Read config values (`get`, `path`, `show`) |
| [`milady dashboard`](/cli/dashboard) | Open the Control UI in your browser |
| [`milady models`](/cli/models) | Show configured model providers |
| [`milady plugins`](/cli/plugins) | Plugin listing, search, install, and management |
| [`milady update`](/cli/update) | Check for and install updates |
| [`milady doctor`](/cli/doctor) | Health and environment checks |
| [`milady db`](/cli/db) | Database utilities (reset) |
| `milady benchmark` | Run a benchmark task headlessly against the agent |

## Global options

| Flag | Description |
|------|-------------|
| `-v, --version` | Print version |
| `--verbose` | Enable informational runtime logs |
| `--debug` | Enable debug-level runtime logs |
| `--dev` | Dev profile: isolate state under `~/.eliza-dev` with separate config and ports |
| `--profile <name>` | Use a named profile (isolates state and config under `~/.eliza-<name>`) |
| `--no-color` | Disable ANSI colors |

### Config subcommands

```bash
milady config get <key>   # read a config value
milady config path        # print resolved config file path
milady config show        # display all config values grouped by section
```

### Plugin subcommands

```bash
milady plugins list            # browse registry plugins
milady plugins installed       # show installed plugins
milady plugins install <name>  # install a plugin from the registry
milady plugins uninstall <name>  # remove an installed plugin
milady plugins search <query>  # search plugins by keyword
```

## Global flags

| Flag | Purpose |
|------|---------|
| `--help` | Show help for any command |
| `--version` | Print the installed version |
| `--verbose` | Enable informational runtime logs |
| `--debug` | Enable debug-level runtime logs |

```bash
milady --version
milady --debug start
milady plugins --help
```

## Help

Run `milady --help` or `milady <command> --help` for the exact flags in your build. Implementation lives under `eliza/packages/app-core/src/cli/` in the elizaOS submodule (run `bun run setup:upstreams` to initialize it for development).
