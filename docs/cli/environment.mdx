# CLI environment variables

These variables are read by the CLI wrapper (`run-node.mjs`) and the embedded runtime. They are **not** a complete list of every model or connector secret â€” see [Configuration](/configuration) and [Plugin setup](/plugin-setup-guide) for provider keys.

## State and config

| Variable | Purpose |
|----------|---------|
| `MILADY_STATE_DIR` | Override state directory (config, credentials, caches). |
| `MILADY_CONFIG_PATH` | Explicit path to `milady.json`. |
| `ELIZA_NAMESPACE` | Namespace segment for default paths (`milady` â†’ `~/.milady/`). |
| `MILADY_WORKSPACE_DIR` / `ELIZA_WORKSPACE_DIR` | Default agent workspace root. |

## Ports and networking

| Variable | Default | Purpose |
|----------|---------|---------|
| `MILADY_API_PORT` / `ELIZA_PORT` | `31337` | HTTP API bind port (dev). In production, the API runs on port 2138. |
| `MILADY_PORT` | `2138` | Dashboard / Vite dev port in orchestrated dev flows. |
| `MILADY_GATEWAY_PORT` | `18789` | Gateway WebSocket + HTTP port. |
| `MILADY_HOME_PORT` | `2142` | Home dashboard port. |
| `MILADY_WECHAT_WEBHOOK_PORT` | `18790` | WeChat webhook listener port. |

## Development

| Variable | Purpose |
|----------|---------|
| `MILADY_SKIP_LOCAL_UPSTREAMS` | Prefer npm `@elizaos/*` over repo-linked `./eliza` packages. |
| `NODE_PATH` | Must include the repo `node_modules` root for dynamic plugin imports â€” set by supported launchers. |

Run `milady doctor` after changing ports or state paths to catch mismatches before you start the server.

## Runtime behavior

| Variable | Purpose |
|----------|---------|
| `MILADY_DISABLE_EDGE_TTS` / `ELIZA_DISABLE_EDGE_TTS` | Set to `1` to prevent auto-loading `@elizaos/plugin-edge-tts`. When the agent orchestrator is loaded, Milady adds Edge TTS by default so swarm/PTY paths have a TTS handlerâ€”this makes outbound calls to Microsoft's cloud. Disable if you want no network TTS. |
| `MILADY_DISABLE_AUTO_BOOTSTRAP` | Set to `1` to skip the one-shot native optimization bootstrap at runtime boot. |
| `EXECUTECODE_TIMEOUT_MS` | Default script timeout for the `EXECUTE_CODE` action, in ms. Default: `30000`. |
| `EXECUTECODE_DISABLE` | Set to `true` or `1` to disable the `EXECUTE_CODE` action entirely. |
| `ELIZA_DISABLE_TRAJECTORY_LOGGING` | Set to `1` to opt out of trajectory persistence. Also disabled when `NODE_ENV=test`. |
