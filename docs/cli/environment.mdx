# CLI environment variables

These variables are read by the CLI wrapper (`run-node.mjs`) and the embedded runtime. They are **not** a complete list of every model or connector secret -- see [Configuration](/configuration) and [Plugin setup](/plugin-setup-guide) for provider keys.

`MILADY_*` variables are aliased to their `ELIZA_*` equivalents at startup (e.g. `MILADY_API_PORT` maps to `ELIZA_API_PORT`). Either form works; the `MILADY_` prefix is the canonical user-facing name.

## State and config

| Variable | Purpose |
|----------|---------|
| `MILADY_STATE_DIR` / `ELIZA_STATE_DIR` | Override state directory (config, credentials, caches). Default `~/.milady/`. |
| `MILADY_CONFIG_PATH` / `ELIZA_CONFIG_PATH` | Explicit path to `milady.json`. |
| `ELIZA_NAMESPACE` | Namespace segment for default paths (`milady` produces `~/.milady/`). |
| `MILADY_WORKSPACE_DIR` / `ELIZA_WORKSPACE_DIR` | Default agent workspace root. |

## Ports and networking

| Variable | Default | Purpose |
|----------|---------|---------|
| `MILADY_PORT` / `ELIZA_UI_PORT` | 2138 | Server-only port (`milady start`) and UI port in dev mode. |
| `MILADY_API_PORT` / `ELIZA_API_PORT` | 31337 | API port in dev mode (when API and UI run on separate ports). |
| `MILADY_API_TOKEN` / `ELIZA_API_TOKEN` | (none) | API authentication token. When set, all API requests require `Authorization: Bearer <token>`. |
| `MILADY_API_BIND` / `ELIZA_API_BIND` | `127.0.0.1` | API server bind address. Non-loopback addresses auto-generate a connection key if no token is set. |
| `MILADY_GATEWAY_PORT` | 18789 | Gateway port (local services). |
| `MILADY_HOME_PORT` | 2142 | Home app port. |

In dev mode (`bun run dev`), the dev orchestrator auto-shifts to the next free port when defaults are busy, and syncs env vars.

## Development

| Variable | Purpose |
|----------|---------|
| `MILADY_SKIP_LOCAL_UPSTREAMS` | Prefer npm `@elizaos/*` over repo-linked `./eliza` packages. |
| `NODE_PATH` | Must include the repo `node_modules` root for dynamic plugin imports -- set by supported launchers. |

Run `milady doctor` after changing ports or state paths to catch mismatches before you start the server.

## Runtime behavior

| Variable | Purpose |
|----------|---------|
| `MILADY_DISABLE_EDGE_TTS` / `ELIZA_DISABLE_EDGE_TTS` | Set to `1` to prevent auto-loading `@elizaos/plugin-edge-tts`. When the agent orchestrator is loaded, Milady adds Edge TTS by default so swarm/PTY paths have a TTS handler -- this makes outbound calls to Microsoft's cloud. Disable if you want no network TTS. |
| `MILADY_DISABLE_AUTO_BOOTSTRAP` | Set to `1` to skip the one-shot native optimization bootstrap at runtime boot. |
| `EXECUTECODE_TIMEOUT_MS` | Default script timeout for the `EXECUTE_CODE` action, in ms. Default: `30000`. |
| `EXECUTECODE_DISABLE` | Set to `true` or `1` to disable the `EXECUTE_CODE` action entirely. |
| `ELIZA_DISABLE_TRAJECTORY_LOGGING` | Set to `1` to opt out of trajectory persistence. Also disabled when `NODE_ENV=test`. |
| `MILADY_ENABLE_CHILD_SKILL_CALLBACK` | Set to `0` to disable the child→parent `USE_SKILL` bridge for spawned coding agents. Default: enabled. |
| `ATROPOS_DATA_DIR` | Staging directory used by the atropos training backend for dataset files. |
| `ATROPOS_BIN` | Override path to the atropos binary when dispatching to that backend. |
