**Milady** est une application d'agent IA client/serveur construite sur [elizaOS](https://github.com/elizaOS).
Le client Milady peut créer un serveur local sur votre machine, découvrir un serveur
sur votre LAN, se connecter à un gateway distant existant ou s'attacher à Eliza Cloud.
Une fois connecté, le serveur sélectionné gère le routage des fournisseurs, les outils, la mémoire et
l'état du runtime via le plan de contrôle du Gateway.

Ce site Mintlify documente le côté développeur et opérateur de Milady :
architecture, configuration, APIs, plugins et comportement du runtime. La documentation
développeur est sur [docs.milady.ai](https://docs.milady.ai). La documentation
consommateur est sur [milady.ai/docs](https://milady.ai/docs).

Milady prend en charge l'inférence locale et les fournisseurs de modèles tiers, dispose d'un écosystème
de plugins en croissance alimenté par elizaOS, d'une marketplace de compétences via
[ClawHub](https://clawhub.ai), et fonctionne sur toutes les plateformes principales : bureau,
mobile, web, CLI et extension Chrome.

<div id="why-milady">
## Pourquoi Milady
</div>

<div id="clientserver-by-default">
### Client/Serveur par défaut
</div>

Milady peut créer un serveur local sur votre propre matériel, découvrir un serveur sur
votre LAN, ou se connecter à un serveur distant de confiance ou Eliza Cloud sans
changer le modèle client. Le serveur sélectionné gère le routage des fournisseurs,
les conversations, les outils et l'état du runtime.

</div>

<div id="plugin-ecosystem">
### Écosystème de plugins
</div>

Milady est livré avec plus de 25 plugins elizaOS de première partie couvrant la gestion des connaissances, l'automatisation du navigateur, l'utilisation de l'ordinateur, Discord, Telegram, TTS (Edge TTS et ElevenLabs), le traitement PDF, l'accès shell, la planification cron, le scoring de confiance et plus encore. Les plugins tiers s'installent en une seule commande via le gestionnaire de plugins.

<div id="skills-marketplace">
### Marketplace de compétences
</div>

Les compétences sont des extensions basées sur markdown qui enseignent à votre agent de nouvelles capacités sans écrire de code. Parcourez et installez des compétences depuis [ClawHub](https://clawhub.ai), le registre de compétences par défaut, ou pointez vers votre propre registre.

<div id="autonomous-mode">
### Mode autonome
</div>

Milady peut fonctionner de manière autonome — exécutant des tâches planifiées, des flux de travail multi-étapes et maintenant sa propre boucle de réflexions et d'actions. Configurez des déclencheurs (intervalle, cron ou ponctuel) et laissez l'agent travailler pour vous.

<div id="built-in-crypto-wallet">
### Portefeuille crypto intégré
</div>

Le flux d'intégration peut générer ou importer des portefeuilles EVM et Solana. Les clés sont stockées localement dans votre configuration et disponibles pour les plugins à l'exécution pour les interactions on-chain, le trading et la DeFi.

<div id="architecture-at-a-glance">
## Architecture en un coup d'œil
</div>

Milady est structuré comme un système en couches :

```
 Interfaces utilisateur (Bureau / Mobile / Web / CLI)
                          |
                     Serveur API (HTTP + WebSocket sur :2138)
                          |
                    Gateway (:18789)
                          |
                   Runtime elizaOS
                     |         |
             Plugin Milady    Autres plugins
             (personnalité,    (connaissances, navigateur,
              portefeuille,     TTS, Discord, Telegram,
              compétences,      shell, cron, ...)
              état autonome)
                     |
               Couche de connecteurs
          (Discord, Telegram, Slack, WhatsApp, Twitter,
           Signal, Farcaster, WeChat, Matrix, Nostr, ...)
```

- **Runtime elizaOS** — le framework central de l'agent qui gère la mémoire, le traitement des messages, le dispatch des actions et l'abstraction des fournisseurs.
- **Plugin Milady** — un plugin elizaOS de première partie qui fournit le contexte du workspace, les clés de session, les presets de personnalité, l'état autonome et l'intégration du portefeuille.
- **Gateway** — un serveur multiplexé WebSocket + HTTP qui relie le runtime à tous les clients et connecteurs connectés.
- **Système de plugins** — plugins elizaOS standard qui ajoutent des capacités (connaissances, navigateur, TTS, etc.).
- **Couche de connecteurs** — adaptateurs spécifiques aux plateformes pour Telegram, Discord, Slack, WhatsApp, Signal, iMessage, Blooio, MS Teams, Google Chat, Twitter, Farcaster, Twitch, Mattermost, WeChat, Matrix, Feishu/Lark, Nostr, Lens et l'interface de chat web.

La configuration principale persistante du runtime se trouve dans `~/.milady/milady.json`.
Le routage canonique du runtime est réparti entre `deploymentTarget`,
`linkedAccounts` et `serviceRouting`. Les secrets peuvent aussi provenir de
`~/.milady/.env`.

<Info>
Milady est actuellement en alpha (`v2.0.0-alpha.116`). Les APIs et la configuration peuvent changer entre les versions. Consultez les [versions GitHub](https://github.com/milady-ai/milady/releases) pour la dernière version.
</Info>
