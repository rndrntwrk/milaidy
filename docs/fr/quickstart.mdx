Au premier lancement, Milady demande :

1. où le runtime doit résider
2. quel personnage utiliser
3. quel fournisseur doit répondre aux messages

<div id="1-pick-the-runtime-location">

## 1. Choisir l'emplacement du runtime

</div>

| Option | Quand l'utiliser |
|--------|------------------|
| `Local` | Milady doit fonctionner sur cette machine (recommandé pour la plupart des utilisateurs) |
| `LAN` | le runtime existe déjà sur une autre machine de votre réseau local |
| `Remote` | le runtime existe déjà sur un VPS, un serveur domestique ou une URL privée |
| `Eliza Cloud` | vous souhaitez un backend géré sans configuration locale |

Si vous n'êtes pas sûr, choisissez `Local`. Vous pourrez changer plus tard — voir [Connexion à distance](/fr/user/connect-remotely).

<div id="2-pick-the-character">

## 2. Choisir le personnage

</div>

Choisissez un nom et un personnage de départ. Cela définit la personnalité, la voix et l'avatar initiaux.

Vous pouvez tout modifier plus tard dans [Changer votre personnage](/fr/user/change-character).

<div id="3-connect-a-provider">

## 3. Connecter un fournisseur

</div>

Choisissez l'une de ces options :

- `Cloud API` — utiliser un fournisseur LLM externe (Anthropic, OpenAI, OpenRouter, Groq, etc.) avec votre propre clé API
- `Ollama` — exécuter un modèle local sur votre machine sans clé API
- `Eliza Cloud` — inférence gérée sans configuration locale

Milady teste la connexion avant de continuer.

<div id="4-send-a-test-message">

## 4. Envoyer un message de test

</div>

Après la configuration, envoyez un court message et vérifiez :

- l'agent répond
- la vitesse est acceptable
- le nom et le ton sont corrects
- la voix fonctionne si vous souhaitez activer la parole
