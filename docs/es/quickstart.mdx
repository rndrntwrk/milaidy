En el primer inicio, Milady pregunta:

1. dónde debe ejecutarse el runtime
2. qué personaje usar
3. qué proveedor debe responder los mensajes

<div id="1-pick-the-runtime-location">

## 1. Elige la ubicación del runtime

</div>

| Opción | Cuándo usarla |
|--------|---------------|
| `Local` | Milady debe ejecutarse en esta máquina (recomendado para la mayoría de usuarios) |
| `LAN` | el runtime ya existe en otra máquina de tu red local |
| `Remote` | el runtime ya existe en un VPS, servidor doméstico o URL privada |
| `Eliza Cloud` | quieres un backend gestionado sin configuración local |

Si no estás seguro, elige `Local`. Puedes cambiarlo después — consulta [Conectar de forma remota](/es/user/connect-remotely).

<div id="2-pick-the-character">

## 2. Elige el personaje

</div>

Elige un nombre y un personaje inicial. Esto establece la personalidad, la voz y el avatar iniciales.

Puedes cambiarlo todo después en [Cambiar tu personaje](/es/user/change-character).

<div id="3-connect-a-provider">

## 3. Conectar un proveedor

</div>

Elige una de estas opciones:

- `Cloud API` — usar un proveedor LLM externo (Anthropic, OpenAI, OpenRouter, Groq, etc.) con tu propia clave API
- `Ollama` — ejecutar un modelo local en tu máquina sin clave API
- `Eliza Cloud` — inferencia gestionada sin configuración local

Milady prueba la conexión antes de continuar.

<div id="4-send-a-test-message">

## 4. Enviar un mensaje de prueba

</div>

Después de la configuración, envía un mensaje corto y confirma:

- que el agente responde
- que la velocidad es aceptable
- que el nombre y el tono son correctos
- que la voz funciona si quieres tener el habla activada
