<div id="prerequisites">
## Requisitos previos
</div>

Antes de instalar Milady, asegúrate de tener el runtime requerido:

| Requisito | Versión | Cómo verificar | Instalar |
|-----------|---------|----------------|----------|
| **Node.js** | >= 22 | `node --version` | [nodejs.org](https://nodejs.org) o `nvm install 22` |

Para **compilar desde el código fuente** o desarrollo, también necesitas:

| Requisito | Versión | Cómo verificar | Instalar |
|-----------|---------|----------------|----------|
| **Bun** | última | `bun --version` | `curl -fsSL https://bun.sh/install \| bash` |

<Info>
Se requiere Node 22+ porque Milady utiliza módulos ES nativos, el prefijo de protocolo `node:` y otras APIs modernas de Node. Bun solo es necesario para desarrollo y compilación desde el código fuente -- se usa como gestor de paquetes y ejecutor de scripts. El paquete npm publicado y los scripts de instalación se ejecutan solo con Node.js.
</Info>

También necesitarás una clave API de al menos un proveedor de modelos (Anthropic, OpenAI, OpenRouter, etc.) -- a menos que planees usar [Ollama](https://ollama.ai) para inferencia completamente local, que no requiere clave API.

<div id="desktop-app-recommended">
## Aplicación de escritorio (Recomendada)
</div>

La aplicación de escritorio es la forma más fácil de comenzar. Empaqueta el runtime completo de Milady, el panel web y el gateway en una aplicación nativa de Electrobun con actualizaciones automáticas e integración con la bandeja del sistema.

Descarga desde **[GitHub Releases](https://github.com/milady-ai/milady/releases/latest)**:

| Plataforma | Archivo | Notas |
|------------|---------|-------|
| macOS (Apple Silicon) | [`Milady-arm64.dmg`](https://github.com/milady-ai/milady/releases/latest) | Macs M1/M2/M3/M4 |
| macOS (Intel) | [`Milady-x64.dmg`](https://github.com/milady-ai/milady/releases/latest) | Macs anteriores a 2020 |
| Windows | [`Milady-Setup.exe`](https://github.com/milady-ai/milady/releases/latest) | Windows 10+ |
| Linux | [`Milady.AppImage`](https://github.com/milady-ai/milady/releases/latest) / [`.deb`](https://github.com/milady-ai/milady/releases/latest) | AppImage o paquete Debian |
| iOS | [App Store](https://apps.apple.com/app/milady-private-ai-assistant/id0000000000) | iPhone y iPad |
| Android | [Google Play](https://play.google.com/store/apps/details?id=ai.milady.app) / [APK](https://github.com/milady-ai/milady/releases/latest) | Android 10+ |

Todas las versiones están firmadas y notarizadas. Sin advertencias de Gatekeeper en macOS.

<div id="verify-download-integrity">
### Verificar la integridad de la descarga
</div>

```bash
cd ~/Downloads
curl -fsSLO https://github.com/milady-ai/milady/releases/latest/download/SHA256SUMS.txt
shasum -a 256 --check --ignore-missing SHA256SUMS.txt
```

<div id="cli-install">
## Instalación por CLI
</div>

<Tabs>
  <Tab title="macOS / Linux">
    El script de instalación descarga la última versión, coloca el binario en tu PATH y ejecuta la configuración inicial:

    ```bash
    curl -fsSL https://milady-ai.github.io/milady/install.sh | bash
    milady setup
    ```

    Luego inicia Milady:

    ```bash
    milady
    ```
  </Tab>
  <Tab title="Windows (PowerShell)">
    ```powershell
    irm https://milady-ai.github.io/milady/install.ps1 | iex
    ```

    Después de la instalación, abre una nueva ventana de terminal y ejecuta:

    ```powershell
    milady setup
    milady
    ```
  </Tab>
  <Tab title="Bun">
    Si tienes Bun instalado:

    ```bash
    bun install -g miladyai
    milady setup
    milady
    ```
  </Tab>
  <Tab title="npm">
    Si prefieres npm:

    ```bash
    npm install -g miladyai
    milady setup
    milady
    ```
  </Tab>
  <Tab title="Homebrew">
    ```bash
    brew tap milady-ai/milady
    brew install milady          # CLI
    brew install --cask milady   # Desktop app (macOS only)
    ```
  </Tab>
  <Tab title="Snap">
    ```bash
    sudo snap install milady
    milady setup
    ```

    Disponible en Ubuntu, Fedora, Manjaro y cualquier distribución con [snapd](https://snapcraft.io/docs/installing-snapd). Los paquetes Snap se actualizan automáticamente en segundo plano.

    Para las últimas versiones de desarrollo:
    ```bash
    sudo snap install milady --edge
    ```
  </Tab>
  <Tab title="Flatpak">
    ```bash
    flatpak install flathub ai.milady.Milady
    flatpak run ai.milady.Milady
    ```

    O instala manualmente desde un [paquete de lanzamiento](https://github.com/milady-ai/milady/releases/latest):
    ```bash
    flatpak --user install milady.flatpak
    ```
  </Tab>
  <Tab title="APT (Debian / Ubuntu)">
    ```bash
    # Add the repository
    curl -fsSL https://apt.milady.ai/gpg.key | sudo gpg --dearmor -o /usr/share/keyrings/milady.gpg
    echo "deb [signed-by=/usr/share/keyrings/milady.gpg] https://apt.milady.ai stable main" | \
      sudo tee /etc/apt/sources.list.d/milady.list

    # Install
    sudo apt update && sudo apt install milady
    ```

    Funciona en Debian 12+, Ubuntu 22.04+, Linux Mint 22+, Pop!_OS y otros derivados de Debian. Las actualizaciones llegan a través de `apt upgrade`.
  </Tab>
</Tabs>

<div id="build-from-source">
## Compilar desde el código fuente
</div>

Para colaboradores o cualquier persona que desee la última versión de desarrollo:

<Steps>
  <Step title="Clonar el repositorio">
    ```bash
    git clone https://github.com/milady-ai/milady.git
    cd milady
    ```
  </Step>
  <Step title="Instalar dependencias">
    ```bash
    bun install
    ```

    El script `postinstall` inicializa automáticamente los submódulos de git, asegura que el directorio de habilidades exista, vincula el servidor del navegador y aplica parches a las dependencias.
  </Step>
  <Step title="Compilar todo">
    ```bash
    bun run build
    ```

    Esto compila el código fuente TypeScript con `tsdown`, escribe la información de compilación y construye la aplicación web en `apps/app`.

    <Info>
    `bun run build:node` es un alias de `bun run build` — ambos ejecutan el mismo script de compilación de producción a través de Node.
    </Info>
  </Step>
  <Step title="Iniciar Milady">
    ```bash
    bun run start
    ```

    Esto lanza el gateway y el servidor API. En la primera ejecución, se te guiará a través del flujo de incorporación interactivo.
  </Step>
</Steps>

<div id="development-mode">
## Modo de desarrollo
</div>

Milady tiene varios modos de desarrollo dependiendo de en qué estés trabajando:

<CodeGroup>
```bash Full dev (runtime + UI hot reload)
bun run dev
```

```bash UI only (web dashboard hot reload)
bun run dev:ui
```

```bash Desktop app (Electrobun with live reload)
bun run dev:desktop
```

```bash Desktop app + Vite HMR
bun run dev:desktop:watch
```
</CodeGroup>

El script `dev` utiliza `scripts/dev-ui.mjs` para iniciar el runtime con recarga en caliente tanto para el backend como para la interfaz web. El script `dev:desktop` inicia la aplicación Electrobun con recarga en vivo.

<div id="environment-setup">
## Configuración del entorno
</div>

Milady almacena todo el estado en `~/.milady/` por defecto. El asistente de incorporación maneja la configuración inicial, pero también puedes establecer variables de entorno directamente.

<div id="api-keys">
### Claves API
</div>

Establece al menos una clave de proveedor de modelos. Puedes hacerlo a través del asistente de incorporación, en `~/.milady/milady.json` bajo el campo `env`, o en `~/.milady/.env`:

<CodeGroup>
```bash Anthropic (recommended)
export ANTHROPIC_API_KEY="sk-ant-..."
```

```bash OpenAI
export OPENAI_API_KEY="sk-..."
```

```bash OpenRouter
export OPENROUTER_API_KEY="sk-or-..."
```

```bash Google Gemini
export GOOGLE_API_KEY="AI..."
```

```bash Ollama (local, no key needed)
export OLLAMA_BASE_URL="http://localhost:11434"
```
</CodeGroup>

<div id="ports">
### Puertos
</div>

| Servicio | Puerto por defecto | Variable de entorno |
|----------|-------------------|---------------------|
| Dashboard + API | `2138` | `MILADY_PORT` |
| Gateway (WebSocket) | `18789` | `MILADY_GATEWAY_PORT` |
| WeChat Webhook | `18790` | `MILADY_WECHAT_WEBHOOK_PORT` |
| Home Dashboard | `2142` | `MILADY_HOME_PORT` |

En modo de desarrollo (`bun run dev`), la API se ejecuta en un puerto separado:

| Servicio | Puerto por defecto | Variable de entorno |
|----------|-------------------|---------------------|
| API + WebSocket (dev) | `31337` | `MILADY_API_PORT` |
| Dashboard UI (dev) | `2138` | `MILADY_PORT` |

```bash
# Example: custom ports
MILADY_PORT=3000 MILADY_GATEWAY_PORT=19000 milady start
```

<div id="state-directory">
### Directorio de estado
</div>

| Variable | Por defecto | Descripción |
|----------|-------------|-------------|
| `MILADY_STATE_DIR` | `~/.milady` | Directorio raíz para todo el estado de Milady |
| `MILADY_CONFIG_PATH` | `~/.milady/milady.json` | Ruta explícita del archivo de configuración |

<div id="security-api-token">
## Seguridad: Token API
</div>

El servidor API se vincula a `127.0.0.1` (loopback) por defecto -- solo tu máquina local puede acceder a él. Si expones el servidor a la red (por ejemplo, `MILADY_API_BIND=0.0.0.0` para despliegues en contenedores o en la nube), **debes** establecer un token de autenticación:

```bash
echo "MILADY_API_TOKEN=$(openssl rand -hex 32)" >> ~/.milady/.env
```

<Warning>
Sin un token en una dirección de enlace pública, cualquier persona que pueda alcanzar el servidor obtiene acceso completo al panel, los endpoints de agentes y las operaciones de billetera.
</Warning>

<div id="verify-installation">
## Verificar la instalación
</div>

Después de la instalación, confirma que todo está funcionando:

```bash
# Check version
milady --version

# Show configured model providers
milady models

# Run the interactive setup wizard
milady setup
```

El comando `models` lista todos los proveedores de modelos configurados y su estado. El comando `setup` ejecuta el asistente de incorporación si es necesario y valida tu espacio de trabajo.

<div id="troubleshooting">
## Solución de problemas
</div>

<div id="node-version-too-old">
### Versión de Node demasiado antigua
</div>

```
Error: Milady requires Node.js >= 22
```

Actualiza Node.js: `nvm install 22 && nvm use 22`, o descárgalo desde [nodejs.org](https://nodejs.org).

<div id="bun-not-found">
### Bun no encontrado
</div>

```
Error: bun: command not found
```

Instala Bun:
```bash
curl -fsSL https://bun.sh/install | bash
```

Luego reinicia tu terminal o ejecuta `source ~/.bashrc` / `source ~/.zshrc`.

<div id="port-already-in-use">
### Puerto en uso
</div>

```
Error: EADDRINUSE :::2138
```

Otro proceso está usando el puerto por defecto. Detenlo o usa un puerto diferente:
```bash
MILADY_PORT=3000 milady start
```

<div id="permission-denied-on-install-script">
### Permiso denegado en el script de instalación
</div>

```bash
chmod +x ~/.milady/bin/milady
```

<div id="pglite-database-corruption">
### PGLite / corrupción de base de datos
</div>

Si ves errores de base de datos al iniciar, Milady intentará una recuperación automática. Si eso falla, puedes restablecer la base de datos:

```bash
rm -rf ~/.milady/workspace/.eliza/.elizadb
milady start
```

<Warning>
Eliminar el directorio de la base de datos borra todo el historial de conversaciones y la memoria. Usa esto como último recurso.
</Warning>

<div id="macos-gatekeeper-blocks-the-app">
### macOS Gatekeeper bloquea la aplicación
</div>

La aplicación de escritorio está firmada y notarizada. Si Gatekeeper aún la bloquea:

```bash
xattr -cr /Applications/Milady.app
```

<div id="local-macos-desktop-builds-unsigned-no-apple-certs">
### Compilaciones locales de escritorio en macOS (sin firmar / sin certificados de Apple)
</div>

Los **lanzamientos** de GitHub están firmados y notarizados. Si **empaquetas la aplicación Electrobun localmente** sin credenciales de Developer ID, establece **`ELECTROBUN_SKIP_CODESIGN=1`** (o deja los secretos sin configurar en CI) para que la compilación no espere la firma de Apple.

En **macOS**, después de `node scripts/desktop-build.mjs package` (desde la raíz del repositorio), el constructor de escritorio ejecuta **`apps/app/electrobun/scripts/local-adhoc-sign-macos.ts`** sobre el **`.app`** producido para que el paquete esté **firmado ad hoc** para pruebas locales. También puedes ejecutarlo manualmente:

```bash
cd apps/app/electrobun && bun scripts/local-adhoc-sign-macos.ts /path/to/Milady.app
```

La firma ad hoc **no** es notarización; Gatekeeper puede requerir **Clic derecho → Abrir** la primera vez. Contexto completo del pipeline: [Compilación y lanzamiento](/es/build-and-release.md).
