Milady supports desktop installs and a CLI runtime. Both use the same local state directory.

## Desktop app

Download the latest stable release from [GitHub Releases](https://github.com/milady-ai/milady/releases/latest).

| Platform | File |
|----------|------|
| macOS (Apple Silicon) | `Milady-arm64.dmg` |
| macOS (Intel) | `Milady-x64.dmg` |
| Windows | `Milady-Setup.exe` |
| Linux | `Milady.AppImage` or `.deb` |
| iOS | App Store |
| Android | Google Play or APK |

### macOS

1. Download the `.dmg`.
2. Drag `Milady.app` into `Applications`.
3. Open it and continue to [First Launch](/quickstart).

### Windows

1. Download `Milady-Setup.exe`.
2. Run the installer.
3. Open Milady from the Start menu.

### Linux

1. Download the `.AppImage` or `.deb`.
2. For AppImage, run `chmod +x Milady.AppImage`.
3. Open the app and continue to [First Launch](/quickstart).

<Info>
Use the official release page. If the app looks unsigned or unfamiliar, delete it and download it again from the project release page.
</Info>

## CLI install

### macOS / Linux

```bash
curl -fsSL https://milady-ai.github.io/milady/install.sh | bash
milady setup
```

### Windows PowerShell

```powershell
irm https://milady-ai.github.io/milady/install.ps1 | iex
milady setup
```

### Start or stop the runtime

```bash
milady start
milady stop
```

## Local state

Milady stores data in:

- macOS / Linux: `~/.milady/`
- Windows: `%USERPROFILE%\\.milady\\`

This directory holds config, conversations, uploaded knowledge, logs, and local assets.

## Before first launch

Have one of these ready:

- an API key from a supported provider
- a local Ollama model
- an Eliza Cloud account
