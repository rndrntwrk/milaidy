**Milady** 是一个构建在 [elizaOS](https://github.com/elizaOS) 之上的客户端/服务器 AI 代理应用。
Milady 客户端可以在你的机器上创建本地服务器、发现局域网上的服务器、
连接到现有的远程网关，或接入 Eliza Cloud。
连接后，所选服务器通过 Gateway 控制平面管理提供者路由、工具、内存和
运行时状态。

这个 Mintlify 站点记录了 Milady 的开发者和运维端：
架构、配置、API、插件和运行时行为。开发者
文档位于 [docs.milady.ai](https://docs.milady.ai)。消费者
文档位于 [milady.ai/docs](https://milady.ai/docs)。

Milady 支持本地推理和第三方模型提供者，拥有由 elizaOS 驱动的不断增长的
插件生态系统，通过 [ClawHub](https://clawhub.ai) 提供技能市场，
并在所有主要平台上运行：桌面、移动端、Web、CLI 和 Chrome 扩展。

<div id="why-milady">
## 为什么选择 Milady
</div>

<div id="clientserver-by-default">
### 默认客户端/服务器架构
</div>

Milady 可以在你自己的硬件上创建本地服务器，发现局域网上的服务器，
或连接到受信任的远程服务器或 Eliza Cloud 服务器，无需
更改客户端模型。所选服务器管理提供者路由、
对话、工具和运行时状态。

<div id="multi-model-multi-provider">
### 多模型、多提供者
</div>

连接到 18 个支持的 AI 提供者中的任何一个 — 或使用 Ollama 完全离线运行：

| 提供者 | 你将获得 |
|--------|----------|
| **Anthropic** | Claude（推荐） |
| **OpenAI** | GPT-4o、o1 及其他 |
| **OpenRouter** | 通过单个 API 访问 100+ 个模型 |
| **Google Gemini** | Gemini Pro、Flash 和 Ultra |
| **Google Antigravity** | Google Cloud / Vertex AI 模型 |
| **xAI** | Grok |
| **Groq** | 超快推理（LPU） |
| **DeepSeek** | 推理和代码模型 |
| **Mistral** | Mistral 和 Mixtral 模型 |
| **Together AI** | 开源模型托管 |
| **Cohere** | Command R+ 和嵌入 |
| **Perplexity** | 搜索增强生成 |
| **Qwen** | 阿里巴巴的 Qwen 模型 |
| **MiniMax** | MiniMax 语言模型 |
| **zAI** | Homunculus Labs 自定义端点 |
| **Vercel AI Gateway** | 统一网关访问 |
| **Ollama** | 完全本地、免费、无需 API 密钥 |
| **Eliza Cloud** | 云服务器目标、关联账户和可选推理路由 |

使用 `/model <id>` 在对话中切换提供者。无锁定。

<div id="plugin-ecosystem">
### 插件生态系统
</div>

Milady 附带 25+ 个 elizaOS 第一方插件，涵盖知识管理、浏览器自动化、计算机使用、Discord、Telegram、TTS（Edge TTS 和 ElevenLabs）、PDF 处理、Shell 访问、Cron 调度、信任评分等。第三方插件通过插件管理器一条命令即可安装。

<div id="skills-marketplace">
### 技能市场
</div>

技能是基于 markdown 的扩展，无需编写代码即可教会你的代理新能力。从 [ClawHub](https://clawhub.ai)（默认技能注册表）浏览并安装技能，或指向你自己的注册表。

<div id="autonomous-mode">
### 自主模式
</div>

Milady 可以自主运行 — 执行计划任务、运行多步骤工作流，并维护自己的思考和行动循环。配置触发器（间隔、cron 或一次性）并让代理代你工作。

<div id="built-in-crypto-wallet">
### 内置加密钱包
</div>

入门流程可以生成或导入 EVM 和 Solana 钱包。密钥存储在本地配置中，并在运行时提供给插件用于链上交互、交易和 DeFi。

<div id="architecture-at-a-glance">
## 架构概览
</div>

Milady 构建为分层系统：

```
 用户界面（桌面 / 移动端 / Web / CLI）
                          |
                     API 服务器（HTTP + WebSocket，端口 :2138）
                          |
                    Gateway (:18789)
                          |
                   elizaOS 运行时
                     |         |
             Milady 插件      其他插件
             （个性、          （知识、浏览器、
              钱包、            TTS、Discord、Telegram、
              技能、            Shell、Cron 等）
              自主状态）
                     |
               连接器层
          （Discord、Telegram、Slack、WhatsApp、Twitter、
           Signal、Farcaster、WeChat、Matrix、Nostr 等）
```

- **elizaOS 运行时** — 处理内存、消息处理、操作分发和提供者抽象的核心代理框架。
- **Milady 插件** — 提供工作区上下文、会话密钥、个性预设、自主状态和钱包集成的第一方 elizaOS 插件。
- **Gateway** — 将运行时桥接到所有连接的客户端和连接器的 WebSocket + HTTP 复用服务器。
- **插件系统** — 添加功能（知识、浏览器、TTS 等）的标准 elizaOS 插件。
- **连接器层** — 适用于 Telegram、Discord、Slack、WhatsApp、Signal、iMessage、Blooio、MS Teams、Google Chat、Twitter、Farcaster、Twitch、Mattermost、WeChat、Matrix、Feishu/Lark、Nostr、Lens 和 Web 聊天界面的平台特定适配器。

运行时的主要持久化配置位于 `~/.milady/milady.json`。
规范的运行时路由分布在 `deploymentTarget`、
`linkedAccounts` 和 `serviceRouting` 之间。密钥也可以来自
`~/.milady/.env`。
