# 中文入门：安装、引导与支持

这篇文档给你一条最短可用路径，目标是 10 分钟内完成首轮可用配置，并在出问题时快速提交高质量反馈。

## 1. 安装与启动

1. 安装依赖并启动：

```bash
bun install
bun run dev
```

2. 打开桌面或 Web 界面，进入首次引导。

提示：首次启动慢于后续启动是正常现象。

## 2. 首次引导怎么选

- `Cloud` 路径：适合希望快速完成配置的用户，默认可用托管能力。
- `自带 Key (BYOK)` 路径：适合已有 OpenAI/Anthropic/OpenRouter 等密钥的用户。

建议：先完成一次可用配置，再做高级切换。

## 3. Cloud 与 BYOK 的关系（核心认知）

- 登录 Eliza Cloud 不等于必须使用 Cloud 推理。
- 你可以保持 Cloud 已连接，同时把推理切到 BYOK 提供商。
- 切换提供商或推理路由后，部分改动需要重启才完全生效；界面会提示并自动处理重启流程。

更多细节见：[Cloud 与提供商路由说明](/zh/guides/cloud-provider-routing)

## 4. 常见失败场景

### 浏览器登录失败

- 检查系统默认浏览器是否可正常打开外链。
- 使用界面内的“重试登录”。
- 若仍失败，先改用 API Key 页签完成连接，再回到登录方式排查。

### 已保存但模型不按预期切换

- 先确认当前选中的推理提供商。
- 观察是否出现“需要重启”提示。
- 执行重启后再测试一次。

### 额度异常或调用失败

- 打开 Cloud 设置查看余额。
- 若 Cloud 已连接但你打算走 BYOK，确认云端推理已关闭。
