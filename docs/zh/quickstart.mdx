首次启动时，Milady 会询问以下内容：

1. 运行时应部署在哪里
2. 使用哪个角色
3. 哪个提供商来回复消息

<div id="1-pick-the-runtime-location">

## 1. 选择运行时位置

</div>

| 选项 | 适用场景 |
|--------|-------------|
| `Local` | Milady 应在本机运行（推荐大多数用户使用） |
| `LAN` | 运行时已存在于局域网内的另一台机器上 |
| `Remote` | 运行时已存在于 VPS、家庭实验室机器或私有 URL 上 |
| `Eliza Cloud` | 你需要一个无需本地设置的托管后端 |

如果不确定，请选择 `Local`。你可以稍后更改——参见[远程连接](/zh/user/connect-remotely)。

<div id="2-pick-the-character">

## 2. 选择角色

</div>

选择一个名称和起始角色。这将设置初始的个性、语音和头像。

你可以稍后在[更改角色](/zh/user/change-character)中修改所有设置。

<div id="3-connect-a-provider">

## 3. 连接提供商

</div>

选择以下之一：

- `Cloud API` — 使用外部 LLM 提供商（Anthropic、OpenAI、OpenRouter、Groq 等），需要自己的 API 密钥
- `Ollama` — 在本地运行模型，无需 API 密钥
- `Eliza Cloud` — 托管推理，无需本地配置

Milady 会在继续之前测试连接。

<div id="4-send-a-test-message">

## 4. 发送测试消息

</div>

设置完成后，发送一条简短消息并确认：

- 智能体有响应
- 响应速度可接受
- 名称和语气正确
- 如果你需要语音功能，语音可以正常工作
