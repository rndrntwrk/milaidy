Milady's character is the combination of its name, tone, voice, and avatar.

## Change these fields

1. `Name`
2. `Personality`
3. `Voice`
4. `Avatar`

## Name

This is what the agent calls itself and what appears in the UI.

## Personality

Keep it short and specific. A good personality prompt covers:

- tone
- kinds of help
- explicit dislikes

Example:

```text
You are a concise, practical assistant. Give direct answers first. Be warm but not chatty. Never use emoji. Focus on planning, writing, and technical troubleshooting.
```

## Voice

Pick the voice you want and test the preview before saving. If previews fail, fix the voice provider first.

## Avatar

Use a built-in avatar or upload a `.vrm` file.

## What the character does not do

The character controls style. It does not replace:

- memory
- uploaded knowledge
- plugins or skills
